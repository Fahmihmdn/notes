----WHERE TO PUT SEARCH TERMS
1. HEADLINES <h1>
2. Distribute throughout article --> use variation. Like sing, sung, sang, singing.
3. Use search terms in link text. Applies even the link between two pages in our web.
4. Put in the page title.  
     <title> </title>
5. Put in description (meta tag) use 20 words to encourage people to visit. Inside <head>
    <meta name="Description" content="desc goes there">
6. Keyword meta tag.  Inside <head>. <meta name="Keywords" content="keyword1, keyword2">
7. Focus on creating great content.


----TIPS GOOD SEO:
1. Number of LINKS has coming into it. 
	One way links is more important. Or exchange links also good. Don;t swap links to spammy sites.
2. If we can make others link to us, try to make our SEARCH TERMS in the LINK TEXT.
3. AVOID have DUPLICATE content, even in our web or from others web.
4. Use DESCRIPTIVE NAMES for folders in the site and filename. Like "buy-cameras.html" --> try to use search terms there too.
5. Consider having a SITEMAP page which links to every other page. Beware using flash/javascript for navigation.
6. UPDATE site regularly. Google likes it. Having a blog can help.
7. AVOID LONG WEB PAGES. Better have pages contain 400 words than 1 pages have 4000 words.

A link from one web to another for GOOGLE is like an ENDORSEMENT. 

IF we DON'T WANT to endorse, we have to put NOFOLLOW attribute in the LINK.
	<a href="" rel="nofollow">LINK</a>

------------------------------------MEASURING SUCCESS-----------------------------------------------------
keyword: visitor, sukses, report.

2 ways:
1. Using server logs
2. Using Javascript Tags.

1. Server logs: See the statistics from hosting. We can see what search engine spiders visit our site. How our bandwith is being used
		We can control spiders that consuming more bandwith than we want using ROBOTS.TXT file. But take care for the configuration.
		Software: AWStats (awstats.sourceforge.net) but it might be set up by the hosting company.

2. Using Javascript: Web analytics solution can built using Javascript. Each web page has a small snippet code added to it. The code track the visitor
		     But, it wont work where Javascript isn't supported. We have to add the TAG to every page on website.
		     GOOGLE ANALYTICS is one of the most popular. (google.com/analytics)
		     

METRIC REPRESENTS:
- Hits: Every file download is a hit. Example: 3 pict, 1 css. Will have 5 hit. But, it tells nothing about human behavior. 
- Bandwith: Total size of all the the files you visitor downloaded. Including spiders automated download.  
- Unique Visitors: Number of different people who visits your web. 
- Number of Visits: The number of different occasions on which people visited your site.
- Page/Page Views: Number of different web pages that were viewed.
- Pages/Visit: Average number of pages viewed in a visit.
- Average time on site: Measured in hours:minutes:second
- Bounce Rate: Percentage of people who arrived at one page and left your site at the same page.
- New Visits: Percentage of people visiting the site that analytics doesnt recognize from a previous visit.

=====Other important metrics:=====
- Number of email newsletter subscribers, RSS Subscribers, Registered member/ Twitter follower.
- How many comments, and forum postings yourr site attracts.

----------------------------------------ACCESIBILITY PRINCIPLES ----------------------------------------------

1. Provide alternative text for anything that isn't text. Image/Video
2. Make sure site still works when JavaScrip, Flash are switched off.
3. Give user warned if they have to respond within a certain limit.
4. Use clear and consisten navigation.
5. Make sure website can be used with keyboard.
6. Make it easy to skip straight to the content, if web page have a long list of links or header information.
7. Don't open a pop up window/change window without warning.
8. Use simplest language
9. Give users control how they view the content
10. Make sure any information conveyed by color can be understood in other ways.
11. Make sure enough contrast between foreground and background. If open in black and white screen.
12. Avoid overcomplicated layouts.
13. Use accesibility features in HTML to help user navigate table and forms.
14. Don't use picture of text unless its essential.

Detailed guidance Web Content Accesibility Guideliness (www.w3.org/TR/WCAG20/) atau (www.section508.gov)
Automated accesibility checkers online (http://wave.webaim.org) and (cynthiasays.com)
